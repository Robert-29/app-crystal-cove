Arquitectura y Flujos de la Aplicación: Crystal Cove

Esta documentación técnica describe los flujos de secuencia exactos de las funcionalidades de la plataforma Crystal Cove, integrando el frontend en React Native (Expo) con el backend de Supabase y la pasarela de pagos de Stripe.

1. Autenticación y Gestión de Perfiles

El sistema utiliza Supabase Auth para manejar las identidades, sincronizando automáticamente el ID de usuario seguro con una tabla pública llamada perfiles para almacenar datos personales como el teléfono y el nombre.

sequenceDiagram participant U as Usuario participant AuthContext as App (Auth Context) participant Auth as Supabase Auth participant DB as Base de Datos (perfiles) opt Registro Nuevo U->>AuthContext: Ingresa Email, Password, Nombre, Teléfono AuthContext->>Auth: signUp(email, password) Auth-->>AuthContext: Retorna UUID del Usuario AuthContext->>DB: INSERT INTO perfiles (id, nombre_completo, telefono) DB-->>AuthContext: Confirmación OK end opt Inicio de Sesión U->>AuthContext: Ingresa Email y Password AuthContext->>Auth: signInWithPassword() Auth-->>AuthContext: Retorna JWT (Sesión) AuthContext->>AuthContext: Actualiza estado global y permite acceso end opt Actualizar Perfil U->>AuthContext: Navega a Perfil y edita datos AuthContext->>DB: UPDATE perfiles SET nombre_completo, telefono DB-->>AuthContext: Éxito end

2. Flujo de Reservación y Pasarela de Pagos (Stripe)

Este proceso crítico asegura que la reserva solo cambie a estado confirmada si el pago con Stripe se procesa exitosamente mediante la Edge Function.

sequenceDiagram participant H as Huésped (App) participant DB as Base de Datos participant Func as Edge Function participant Stripe as Stripe API H->>H: Toca "Confirmar Reserva" H->>DB: Verifica sesión y disponibilidad H->>DB: INSERT en reservaciones (estado: 'pendiente') DB-->>H: Retorna objeto Reserva (Incluye ID único) H->>Func: invoke('create-payment-intent', monto, reservaId) Func->>Stripe: paymentIntents.create({amount, currency: mxn}) Stripe-->>Func: Retorna PaymentIntent Object Func-->>H: Retorna {clientSecret} H->>H: initPaymentSheet(clientSecret) H->>H: presentPaymentSheet() (Abre interfaz de tarjeta) alt Pago Cancelado o Fallido H->>H: Muestra Alerta de Error. La reserva se queda 'pendiente'. else Pago Exitoso H->>DB: UPDATE reservaciones SET estado='confirmada', codigo_qr='QR-{id}' DB-->>H: OK H->>H: Redirige a pantalla de Mis Reservas end

3. Escáner QR y Recepción (Administrador)

Flujo nativo que permite a un administrador validar de forma segura un ticket generado y registrar la entrada o cancelación del huésped en las instalaciones.

sequenceDiagram participant A as Administrador participant Cam as Cámara del Móvil participant DB as Base de Datos A->>Cam: Abre módulo de Escáner QR Cam-->>A: Detecta cadena de texto (Ej: "QR-123...") A->>DB: SELECT reservaciones JOIN perfiles WHERE codigo_qr = detectado DB-->>A: Retorna datos completos (Habitación, Nombre, Teléfono, Estado) A->>A: Renderiza 'Ticket de Admin' en Pantalla alt Toca "Marcar En Estancia" A->>DB: UPDATE reservaciones SET estado='en_estancia' WHERE id DB-->>A: Éxito else Toca "Cancelar Reserva" A->>DB: UPDATE reservaciones SET estado='cancelada' WHERE id DB-->>A: Éxito end A->>A: Muestra alerta de éxito y reactiva la cámara

4. Dashboard Analítico (Administrador en Tiempo Real)

El dashboard aprovecha Supabase Realtime (WebSockets) para mantener las gráficas y KPIs actualizados de manera inmediata cuando hay un cambio en la base de datos.

Carga Inicial A->>DB: GET perfiles & reservaciones DB-->>A: Retorna arrays de datos A->>A: Calcula Métricas y Dibuja Gráfica de Anillo (SVG) Note right of A: 2. Suscripción a WebSockets A->>RT: channel('admin_channel').on('postgres_changes') RT-->>A: Conexión Establecida opt Cliente hace nueva reserva DB->>RT: Evento UPDATE / INSERT RT-->>A: Notifica cambios (payload) A->>A: Re-ejecuta cálculos y actualiza el Dashboard sin recargar end
